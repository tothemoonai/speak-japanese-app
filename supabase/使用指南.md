## 创建数据库表

### 方法 1: 使用 Supabase SQL Editor（推荐）

1. **登录 Supabase Dashboard**
   - 访问 [https://supabase.com/dashboard](https://supabase.com/dashboard)
   - 选择你的项目

2. **打开 SQL Editor**
   - 在左侧导航栏点击 "SQL Editor"
   - 点击 "New Query"

3. **执行 schema.sql**
   - 复制 `supabase/schema.sql` 文件的全部内容
   - 粘贴到 SQL Editor 中
   - 点击 "Run" 执行

4. **验证创建结果**
   - 检查 "Table Editor" 标签页
   - 应该能看到以下表：
     - `users`
     - `courses`
     - `characters`
     - `sentences`
     - `practice_records`
     - `practice_results`
     - `daily_reports`
     - `shares`

### 方法 2: 使用 psql 命令行

如果你更喜欢使用命令行：

```bash
# 设置环境变量
export SUPABASE_DB_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT-REF].supabase.co:5432/postgres"

# 执行 schema.sql
psql $SUPABASE_DB_URL -f supabase/schema.sql
```

## 验证设置

### 1. 检查表结构

在 SQL Editor 中运行：

```sql
-- 查看所有表
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

**预期结果**:
```
table_name
------------------
characters
courses
daily_reports
practice_records
practice_results
sentences
shares
users
```

### 2. 检查课程数据

```sql
SELECT id, course_number, title_cn, title_jp, difficulty
FROM courses
ORDER BY course_number;
```

**预期结果**: 8 个课程

### 3. 检查角色数据

```sql
SELECT c.id, co.title_cn, c.name_cn, c.name_jp, c.difficulty_level
FROM characters c
JOIN courses co ON c.course_id = co.id
ORDER BY c.course_id, c.id
LIMIT 10;
```

**预期结果**: 16 个角色（每个课程 2 个角色）

### 4. 检查句子数据

```sql
SELECT id, course_id, sentence_order, text_cn, text_jp
FROM sentences
ORDER BY course_id, sentence_order
LIMIT 10;
```

**预期结果**: 至少 30 个句子

### 5. 检查用户表触发器

```sql
-- 查看触发器
SELECT trigger_name, event_manipulation, event_object_table
FROM information_schema.triggers
WHERE trigger_schema = 'public'
AND trigger_name = 'on_auth_user_created';
```

**预期结果**: 应该有一个触发器

## RLS (Row Level Security) 策略说明

### 自动应用的策略

1. **用户只能查看/修改自己的数据**
   - `users` 表
   - `practice_records` 表
   - `practice_results` 表
   - `daily_reports` 表
   - `shares` 表（只能修改自己创建的）

2. **所有人可以读取公开数据**
   - `courses` 表
   - `characters` 表
   - `sentences` 表

### 测试 RLS 策略

你可以通过 SQL Editor 测试 RLS 策略：

```sql
-- 切换到特定用户上下文（需要真实的 user_id）
SET LOCAL jwt.claims.sub = 'your-user-id-here';

-- 测试用户只能看到自己的练习记录
SELECT * FROM practice_records;
```

## 常见问题

### Q1: 执行 SQL 时出现 "relation does not exist" 错误

**原因**: 表还没有创建

**解决**:
1. 先执行 `schema.sql` 创建所有表
2. 然后执行 `seed-data.sql` 插入测试数据

### Q2: 用户注册后 users 表中没有记录

**原因**: 触发器没有创建或失败

**解决**:
1. 检查触发器是否存在：
   ```sql
   SELECT * FROM pg_trigger WHERE tgname = 'on_auth_user_created';
   ```
2. 如果不存在，重新执行 schema.sql 中的触发器部分
3. 检查 `public.handle_new_user()` 函数是否存在

### Q3: 外键约束错误

**原因**: 数据插入顺序错误

**解决**:
确保按照以下顺序插入数据：
1. courses（课程）
2. characters（角色）
3. sentences（句子）
4. users（用户）
5. practice_records（练习记录）
6. practice_results（练习结果）

### Q4: RLS 策略阻止了所有访问

**原因**: RLS 已启用但没有正确配置策略

**解决**:
1. 临时禁用 RLS：
   ```sql
   ALTER TABLE public.users DISABLE ROW LEVEL SECURITY;
   ```
2. 检查策略配置
3. 重新启用 RLS：
   ```sql
   ALTER TABLE public.users ENABLE ROW LEVEL SECURITY;
   ```

### Q5: 测试数据插入失败

**原因**: 可能是因为依赖的数据不存在

**解决**:
1. 先执行 `schema.sql`
2. 确保所有表都已创建
3. 再执行 `seed-data.sql`

## 验证测试数据完整性

### 运行完整性检查

```sql
-- 检查课程是否有对应的角色
SELECT c.id, c.title_cn
FROM courses c
LEFT JOIN characters ch ON ch.course_id = c.id
WHERE ch.id IS NULL;

-- 检查角色是否有对应的句子
SELECT ch.id, ch.name_cn
FROM characters ch
LEFT JOIN sentences s ON s.character_id = ch.id
WHERE s.id IS NULL;

-- 检查所有句子是否都有有效的课程和角色
SELECT s.id, s.text_cn
FROM sentences s
LEFT JOIN courses c ON s.course_id = c.id
LEFT JOIN characters ch ON s.character_id = ch.id
WHERE c.id IS NULL OR ch.id IS NULL;
```

**预期结果**: 所有查询都应该返回空结果（0 rows）

## 附录：快速参考

### 重要查询

```sql
-- 查看所有课程及其句子数量
SELECT
  c.id,
  c.course_number,
  c.title_cn,
  COUNT(s.id) as sentence_count
FROM courses c
LEFT JOIN sentences s ON s.course_id = c.id
GROUP BY c.id
ORDER BY c.course_number;

-- 查看用户的练习统计
SELECT
  u.nickname,
  u.level,
  COUNT(pr.id) as total_practices,
  AVG(pr.total_score) as avg_score
FROM users u
LEFT JOIN practice_records pr ON pr.user_id = u.id
WHERE pr.completed_at IS NOT NULL
GROUP BY u.id;

-- 查看某个课程的详细内容
SELECT
  co.title_cn as course_title,
  ch.name_cn as character_name,
  s.sentence_order,
  s.text_cn,
  s.text_jp
FROM courses co
JOIN characters ch ON ch.course_id = co.id
JOIN sentences s ON s.character_id = ch.id
WHERE co.id = 1
ORDER BY s.sentence_order;
```
