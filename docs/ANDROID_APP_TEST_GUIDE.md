# Android APP ASR 测试指南

## 📱 安装和测试步骤

### 1. 安装APK

**方法一：使用USB连接**
```bash
# 连接Android设备到电脑，启用USB调试
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

**方法二：直接传输**
```bash
# 将APK文件传输到设备
# 文件位置：android/app/build/outputs/apk/debug/app-debug.apk
# 在设备上直接点击安装
```

### 2. 打开APP并访问测试页面

**步骤**：
1. 打开"日语口语练习"APP
2. 在APP内输入地址栏或使用以下方法之一：

**方法A：直接输入URL**
```
https://speak-japanese-app.vercel.app/test-android-asr
```

**方法B：从首页进入**
1. 访问首页：`https://speak-japanese-app.vercel.app`
2. 滚动到页面底部
3. 点击"Android ASR 测试"链接

### 3. 测试ASR功能

**测试步骤**：

#### 步骤1：初始化ASR
- 点击"初始化 ASR"按钮
- 等待模型加载（首次使用需要较长时间，约10-30秒）
- 看到"就绪"状态表示初始化成功

#### 步骤2：开始录音
- 点击"开始录音"按钮
- 授予麦克风权限（首次使用）
- 按住按钮开始说话

#### 步骤3：停止录音并识别
- 点击"停止录音"按钮
- 等待识别结果显示（约1-3秒）
- 查看识别的文本结果

**建议测试的日语句子**：
- こんにちは (你好)
- ありがとう (谢谢)
- さようなら (再见)
- おはようございます (早上好)
- こんにちは、元気ですか (你好，你好吗)

### 5. 常见问题排查

#### 问题1：显示"平台不兼容"
- **原因**：在Chrome浏览器中打开而非APP
- **解决**：必须在编译的Android APP中打开

#### 问题2：初始化失败
- **可能原因**：
  - 模型文件损坏
  - 权限问题
  - 存储空间不足
- **解决**：
  - 重新安装APK
  - 清理APP缓存
  - 确保有足够的存储空间

#### 问题3：录音失败
- **可能原因**：
  - 麦克风权限未授予
  - 麦克风被其他应用占用
- **解决**：
  - 在设置中授予麦克风权限
  - 关闭其他使用麦克风的应用

#### 问题4：识别结果不准确
- **可能原因**：
  - 环境噪音太大
  - 说话不够清晰
  - 语音过短或过长
- **建议**：
  - 在安静环境测试
  - 清晰地发音
  - 说出完整的句子（3-10秒）

### 6. 性能基准

**预期性能**：
- 初始化时间：10-30秒（首次）
- 后续启动：< 5秒
- 录音响应：< 100ms
- 识别时间：1-3秒
- 内存占用：200-400MB
- APK大小：265MB

**测试设备**：
- 推荐：Android 8.0+
- RAM：至少3GB
- 存储空间：至少1GB可用空间

### 7. 日志收集

**开启调试日志**：
在APP初始化时设置`debug: true`：
```typescript
await asr.initialize({
  language: 'ja',
  debug: true,
});
```

**查看日志**：
```bash
adb logcat | grep -i "sherpa\|asr"
```
