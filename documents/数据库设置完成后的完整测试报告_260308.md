## 🔍 详细失败分析

### 失败 #1: 登录页面缺少 h1 标签

**失败测试**: "navigation works" (5 个浏览器)

**错误信息**:
```
Error: expect(locator).toContainText(expected) failed
Locator: locator('h1')
Expected substring: "登录"
Timeout: 5000ms
Error: element(s) not found
```

**根本原因**:
`app/login/page.tsx` 组件中没有 `<h1>` 标签

**修复方案**:
```tsx
// app/login/page.tsx
export default function LoginPage() {
  return (
    <div className="min-h-screen flex items-center justify-center">
      <div className="max-w-md w-full">
        <h1>登录</h1>  {/* 添加这一行 */}
        {/* ... rest of the form */}
      </div>
    </div>
  );
}
```

**影响**:
- 测试失败率: 14.3% (5/35)
- 用户体验: 轻微（SEO 和可访问性略有影响）

---

### 失败 #2: Safari 表单验证问题

**失败测试**:
- "shows error when passwords do not match" (WebKit, Mobile Safari)
- "shows error when password is too short" (Mobile Safari)

**错误信息**:
```
Error: expect(locator).toBeVisible() failed
Locator: locator('text=两次输入的密码不一致')
Expected: visible
Timeout: 5000ms
Error: element(s) not found
```

**根本原因**:
Safari/WebKit 浏览器的表单验证行为与 Chrome/Firefox 不同，可能是：
1. 表单提交事件处理时机不同
2. 错误状态更新延迟
3. 渲染时机问题

**调查步骤**:
1. 在 Safari 浏览器中手动测试表单验证
2. 检查 `useForm` 或表单处理逻辑
3. 确认错误状态是否正确设置
4. 测试添加延迟等待元素可见

**临时方案**:
在测试中添加显式等待：
```typescript
await page.click('button[type="submit"]');
await page.waitForTimeout(500); // 添加延迟
await expect(page.locator('text=两次输入的密码不一致')).toBeVisible();
```
