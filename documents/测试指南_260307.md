## 🔧 测试环境准备

### 1. 安装项目依赖

在项目根目录运行：

```bash
npm install
```

### 2. 创建测试环境变量文件

```bash
# Windows
copy .env.example .env.test

# Linux/Mac
cp .env.example .env.test
```

编辑 `.env.test` 文件，配置测试环境变量：

```env
# AI服务测试密钥（可选，测试中会被mock）
OPENAI_API_KEY=test-openai-key
ANTHROPIC_API_KEY=test-anthropic-key
ZHIPU_API_KEY=test-zhipu-key

# 数据库测试配置（可选）
NEXT_PUBLIC_SUPABASE_URL=your-test-supabase-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-test-anon-key

# 注意：单元测试会mock所有API调用，不会消耗真实额度
```

### 3. 安装 Playwright 浏览器（首次运行 E2E 测试需要）

```bash
npx playwright install
```


**终端 1**:
```bash

```

## 🔍 故障排查

### 单元测试问题

#### 问题：测试失败，提示 "Cannot find module"

**解决方案**:
```bash
# 确保所有依赖已安装
npm install

# 清理并重新安装
rm -rf node_modules package-lock.json
npm install
```

#### 问题：Mock 函数未正确工作

**解决方案**:
检查 `jest.setup.js` 是否正确配置，确保在测试前清除了 mock：

```javascript
beforeEach(() => {
  jest.clearAllMocks();
});
```

#### 问题：环境变量未加载

**解决方案**:
确保 `.env.test` 文件存在且格式正确，不要包含引号：

```env
# 正确
ZHIPU_API_KEY=test-key

# 错误
ZHIPU_API_KEY="test-key"
```

### E2E 测试问题

#### 问题：测试失败，提示 "Cannot connect to server"

**解决方案**:
1. 确保开发服务器正在运行：`npm run dev`
2. 检查服务器是否在 `http://localhost:3000`
3. 确认 `playwright.config.ts` 中 baseURL 配置正确

#### 问题：浏览器未安装

**解决方案**:
```bash
# 安装所有浏览器
npx playwright install

# 只安装特定浏览器
npx playwright install chromium
```

#### 问题：测试超时

**解决方案**:
在 `playwright.config.ts` 中增加超时时间：

```typescript
use: {
  baseURL: 'http://localhost:3000',
  actionTimeout: 10000,  // 增加到 10 秒
}
```

#### 问题：测试在 CI 环境中失败

**解决方案**:
确保 CI 环境配置了正确的环境变量，并且安装了浏览器依赖：

```bash
# Linux CI 环境可能需要
npx playwright install-deps
```

## 📝 测试最佳实践

### 1. 编写测试前

- [ ] 理解被测试功能的需求
- [ ] 确定测试边界情况
- [ ] 准备测试数据和 mock

### 2. 编写测试时

- [ ] 使用描述性的测试名称
- [ ] 一个测试只验证一件事
- [ ] 使用 beforeEach/afterEach 清理状态
- [ ] Mock 外部依赖（API、数据库等）

### 3. 运行测试后

- [ ] 检查测试覆盖率
- [ ] 查看测试报告
- [ ] 修复失败的测试
- [ ] 重构重复代码

### 4. CI/CD 集成

建议在 CI/CD 流程中包含：

```yaml
# .github/workflows/test.yml 示例
name: Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm install
      - run: npm test
      - run: npm run test:e2e
```

## 🔄 持续集成测试

### GitHub Actions 配置示例

创建 `.github/workflows/test.yml`：

```yaml
name: Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  unit-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npm test -- --coverage
      - uses: codecov/codecov-action@v3

  e2e-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npx playwright install --with-deps
      - run: npm run build
      - run: npm start &
      - run: npm run test:e2e
      - uses: actions/upload-artifact@v3
        if: always()
        with:
          name: playwright-report
          path: playwright-report/
```
