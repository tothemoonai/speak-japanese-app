## 🚀 下一步操作指南

### 方法 1: 使用 Supabase SQL Editor（推荐）

这是最简单、最安全的方法，无需安装额外工具。

#### 步骤：

1. **登录 Supabase Dashboard**
   ```
   访问: https://supabase.com/dashboard
   选择你的项目
   ```

2. **创建数据库表结构**
   - 点击左侧菜单的 "SQL Editor"
   - 点击 "New Query" 创建新查询
   - 复制 `supabase/schema.sql` 文件的全部内容
   - 粘贴到 SQL Editor 中
   - 点击 "Run" 按钮执行
   - 等待完成（约 5-10 秒）

3. **导入测试数据**
   - 创建新的查询（"New Query"）
   - 复制 `supabase/seed-data.sql` 文件的全部内容
   - 粘贴到 SQL Editor 中
   - 点击 "Run" 按钮执行
   - 等待完成（约 5-10 秒）

4. **验证设置**
   - 点击左侧菜单的 "Table Editor"
   - 应该能看到 8 个表：
     * courses (8 条记录)
     * characters (16 条记录)
     * sentences (30+ 条记录)
     * users (空表，等待用户注册)
     * practice_records (空表)
     * practice_results (空表)
     * daily_reports (空表)
     * shares (2 条记录)

### 方法 2: 使用 psql 命令行

如果你已安装 PostgreSQL 客户端：

#### 步骤：

1. **获取数据库密码**
   - 在 Supabase Dashboard 中
   - 进入 Project Settings → Database
   - 复制数据库密码

2. **设置环境变量**
   ```bash
   # Linux/Mac
   export SUPABASE_URL='https://your-project-ref.supabase.co'
   export SUPABASE_ANON_KEY='your-anon-key'

   # Windows PowerShell
   $env:SUPABASE_URL='https://your-project-ref.supabase.co'
   $env:SUPABASE_ANON_KEY='your-anon-key'
   ```

3. **运行设置脚本**
   ```bash
   bash supabase/setup.sh
   ```

4. **输入数据库密码**
   - 脚本会提示输入数据库密码
   - 粘贴刚才复制的密码
   - 等待脚本执行完成

### 方法 3: 手动 psql 命令

```bash
# 设置连接字符串
export SUPABASE_DB_URL="postgresql://postgres:[YOUR-PASSWORD]@db.[YOUR-PROJECT-REF].supabase.co:5432/postgres"

# 执行 schema
psql $SUPABASE_DB_URL -f supabase/schema.sql

# 执行 seed-data
psql $SUPABASE_DB_URL -f supabase/seed-data.sql
```

## 🔧 数据库功能特性

### 1. 自动用户配置创建
当用户在应用中注册时，触发器会自动在 `public.users` 表中创建对应的用户配置记录：

```sql
-- 触发器函数
CREATE FUNCTION handle_new_user()
-- 自动从 email 提取 nickname
-- 自动设置默认 level 为 'beginner'
-- 自动设置时间戳
```

### 2. Row Level Security (RLS)
每个用户只能访问自己的数据：

- ✅ users 表：只能查看/修改自己的配置
- ✅ practice_records 表：只能查看自己的练习记录
- ✅ practice_results 表：只能查看自己的练习结果
- ✅ daily_reports 表：只能查看自己的每日报告
- ✅ shares 表：可以创建/修改自己的分享，但所有人可以查看

公开数据（所有人可读）：
- ✅ courses 表
- ✅ characters 表
- ✅ sentences 表

### 3. JSONB 字段支持
灵活的数据结构：

- `sentences.vocabulary` - 词汇解释 (JSONB)
- `sentences.grammar_points` - 语法点 (JSONB)
- `practice_results.dimension_scores` - 5 维评分 (JSONB)
- `practice_results.feedback` - 反馈信息 (JSONB)
- `daily_reports.progress_chart` - 进度图表 (JSONB)

### 4. 数据完整性约束
- ✅ 外键级联删除（删除课程时自动删除相关角色和句子）
- ✅ CHECK 约束（限制枚举值）
- ✅ UNIQUE 约束（防止重复）
- ✅ NOT NULL 约束（必需字段）

### 5. 性能优化
- ✅ 在常用查询字段上创建索引
- ✅ 使用视图简化复杂查询
- ✅ JSONB 字段支持高效查询

## 🛠️ 常用维护命令

### 重置测试数据

如果需要清空并重新导入测试数据：

**方法 1: 使用脚本**
```bash
bash supabase/reset-data.sh
```

**方法 2: 手动 SQL**
在 SQL Editor 中运行：

```sql
-- 清空所有测试数据
TRUNCATE TABLE public.practice_results CASCADE;
TRUNCATE TABLE public.practice_records CASCADE;
TRUNCATE TABLE public.daily_reports CASCADE;
TRUNCATE TABLE public.shares CASCADE;
TRUNCATE TABLE public.sentences CASCADE;
TRUNCATE TABLE public.characters CASCADE;
TRUNCATE TABLE public.courses CASCADE;

-- 然后重新执行 seed-data.sql
```

### 备份数据

```bash
# 备份所有表数据
pg_dump $SUPABASE_DB_URL --data-only --clean > backup.sql
```

### 恢复数据

```bash
psql $SUPABASE_DB_URL < backup.sql
```
