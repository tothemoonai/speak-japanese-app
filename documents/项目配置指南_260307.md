## 环境准备

### 1. 系统要求

#### 硬件要求
- **CPU**: 双核及以上
- **内存**: 4GB RAM 及以上（推荐 8GB）
- **硬盘**: 至少 2GB 可用空间

#### 操作系统支持
- **Windows**: Windows 10/11
- **macOS**: macOS 10.15+
- **Linux**: Ubuntu 20.04+, Debian 11+, CentOS 8+

**Node.js**

**检查Node.js版本**

```bash
node --version
```

3. 运行安装程序，按提示完成安装

## Supabase配置

Supabase是一个开源的Firebase替代方案，提供数据库、认证、存储等功能。

### 步骤1: 创建Supabase账户

1. **访问Supabase官网**
   - 打开浏览器，访问 https://supabase.com
   - 点击右上角 "Start your project"

2. **注册账户**
   - 使用GitHub账户登录（推荐）
   - 或使用邮箱注册

### 步骤2: 创建新项目

1. **创建组织**
   ```
   - 登录后会提示创建组织（Organization）
   - 输入组织名称，例如: "Personal"
   - 选择组织用途: "Personal" 或 "Education"
   ```

2. **创建项目**
   ```
   - 点击 "New Project"
   - 填写项目信息:
     * Name: speak-japanese-app
     * Database Password: 设置强密码（务必保存）
     * Region: 选择距离最近的区域
       * 中国用户推荐: Singapore (ap-southeast-1)
       * 或选择: Northeast Asia (Tokyo)
   - 点击 "Create new project"
   ```

3. **等待项目创建**
   - 项目创建通常需要 1-2 分钟
   - 创建成功后会进入项目仪表板

### 步骤3: 获取API密钥

1. **进入项目设置**
   ```
   - 在项目仪表板左侧菜单
   - 点击 "Settings" → "API"
   ```

2. **复制以下信息**
   ```
   - Project URL: 类似 https://xxxxx.supabase.co
   - anon public: 公开匿名密钥
   - service_role: 服务端角色密钥（保密！）
   ```

3. **填写到环境变量**
   ```
   # .env.local
   NEXT_PUBLIC_SUPABASE_URL=<Project URL>
   NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon public>
   SUPABASE_SERVICE_ROLE_KEY=<service_role>
   ```

**⚠️ 重要提示**:
- `NEXT_PUBLIC_SUPABASE_URL` 和 `NEXT_PUBLIC_SUPABASE_ANON_KEY` 是公开的，可以暴露在浏览器中
- `SUPABASE_SERVICE_ROLE_KEY` 是敏感信息，只能在服务端使用，切勿提交到Git

### 步骤5: 配置Row Level Security (RLS)

RLS（行级安全）确保用户只能访问自己的数据。

1. **创建RLS策略**
   ```sql
   -- 在SQL编辑器中运行

   -- 启用RLS
   ALTER TABLE users ENABLE ROW LEVEL SECURITY;
   ALTER TABLE practice_records ENABLE ROW LEVEL SECURITY;
   ALTER TABLE sentence_practices ENABLE ROW LEVEL SECURITY;
   ALTER TABLE user_achievements ENABLE ROW LEVEL SECURITY;
   ALTER TABLE shares ENABLE ROW LEVEL SECURITY;

   -- 用户只能查看自己的数据
   CREATE POLICY "Users can view own data" ON users
     FOR SELECT USING (auth.uid() = id);

   CREATE POLICY "Users can insert own practice records" ON practice_records
     FOR INSERT WITH CHECK (auth.uid() = user_id);

   CREATE POLICY "Users can view own practice records" ON practice_records
     FOR SELECT USING (auth.uid() = user_id);

   CREATE POLICY "Users can update own practice records" ON practice_records
     FOR UPDATE USING (auth.uid() = user_id);

   -- 其他表的策略...
   ```

2. **验证RLS**
   ```
   - 在 Table Editor 中点击表名
   - 在右侧栏找到 "RLS policies"
   - 应该能看到刚创建的策略
   ```

   ```sql
   -- 允许公开访问
   CREATE POLICY "Public Access"

   TO anon
   USING (bucket_id = 'audio-files');

   -- 允许认证用户上传
   CREATE POLICY "Auth Upload"
   ON storage.objects FOR INSERT
   TO authenticated
   WITH CHECK (bucket_id = 'audio-files');
   ```

## AI服务配置

AI服务用于智能评估用户的口语。至少需要配置一个。

### 选项1: OpenAI（推荐）

#### 获取API密钥

1. **访问OpenAI官网**
   - 打开 https://platform.openai.com/
   - 使用邮箱或Google账户登录

2. **创建API密钥**
   ```
   - 点击左侧菜单 "API keys"
   - 点击 "+ Create new secret key"
   - 命名密钥: "Speak Japanese App"
   - 复制并保存密钥（只显示一次！）
   ```

3. **配置到环境变量**
   ```env
   OPENAI_API_KEY=sk-proj-your-key-here
   ```

4. **设置付款方式**
   ```
   - OpenAI API是付费服务
   - 需要绑定信用卡或借记卡
   - 新用户有$5免费额度
   - 按使用量付费
   ```

#### 费用估算

使用GPT-4进行评估：
- 每次评估消耗约 500-1000 tokens
- 1M tokens ≈ $30
- 假设每次练习评估10句话，约消耗 5000-10000 tokens
- 100次练习 ≈ $0.15-$0.30

使用GPT-3.5-turbo（更便宜）：
- 每次评估消耗约 500-1000 tokens
- 1M tokens ≈ $2
- 费用约为GPT-4的1/15

### 选项2: Anthropic Claude

#### 获取API密钥

1. **访问Anthropic官网**
   - 打开 https://console.anthropic.com/
   - 使用邮箱注册/登录

2. **创建API密钥**
   ```
   - 点击 "Get API Keys"
   - 点击 "Create Key"
   - 复制并保存密钥
   ```

3. **配置到环境变量**
   ```env
   ANTHROPIC_API_KEY=sk-ant-your-key-here
   ```

4. **设置付款方式**
   ```
   - Claude也是付费服务
   - 需要绑定付款方式
   - 新用户有免费额度
   - 按使用量付费
   ```

#### 费用估算

Claude 3 Haiku（最便宜）:
- 每次评估约 $0.00025
- 1000次练习 ≈ $2.50

### 选项3: 智谱GLM（推荐用于国内测试）

智谱AI的GLM-4模型是国内领先的中文大模型，非常适合用于日语口语评估。

#### 获取API密钥

1. **访问智谱AI开放平台**
   - 打开 https://open.bigmodel.cn/
   - 点击右上角 "注册" 或 "登录"

2. **完成实名认证**
   ```
   - 需要进行个人实名认证
   - 准备身份证照片
   - 认证过程通常需要几分钟
   ```

3. **获取API密钥**
   ```
   - 登录后进入控制台
   - 点击 "API密钥"
   - 点击 "创建新的API密钥"
   - 复制并保存密钥（格式类似：xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx.xx.xxxxxxxx）
   ```

4. **配置到环境变量**
   ```env
   ZHIPU_API_KEY=your-zhipu-api-key-here
   ```

#### 费用说明

智谱GLM-4的定价：
- **免费额度**：新用户有免费试用额度
- **按需付费**：
  - GLM-4：输入 ¥0.50/千tokens，输出 ¥2.00/千tokens
  - GLM-3-Turbo：输入 ¥0.005/千tokens，输出 ¥0.005/千tokens（更便宜）

每次评估约消耗 500-1000 tokens：
- 使用GLM-4：每次评估约 ¥0.25-¥0.50
- 使用GLM-3-Turbo：每次评估约 ¥0.0025-¥0.005

100次练习成本估算：
- GLM-4：约 ¥25-¥50
- GLM-3-Turbo：约 ¥0.25-¥0.50

## 常见问题排查

### 问题1: Node.js版本过低

**症状**:
```
Error: Node.js version too old. Requires 18.17 or higher.
```

**解决方案**:
```bash
# 1. 卸载旧版Node.js

# 2. 下载安装最新LTS版本
# 访问 https://nodejs.org/

# 3. 验证版本
node --version
```

### 问题2: 依赖安装失败

**症状**:
```
npm ERR! code ERESOLVE
npm ERR! ERESOLVE unable to resolve dependency tree
```

**解决方案**:
```bash
# 方案1: 使用 --legacy-peer-deps
npm install --legacy-peer-deps

# 方案2: 清除缓存重试
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 方案3: 使用npx强制安装
npx npm-install-missing
```

### 问题3: Supabase连接失败

**症状**:
```
Error: Failed to fetch: Invalid URL
```

**解决方案**:
1. 检查 `.env.local` 文件是否存在
2. 检查 `NEXT_PUBLIC_SUPABASE_URL` 是否正确
3. 确保URL格式为: `https://xxxxx.supabase.co`
4. 检查网络连接
5. 尝试重启开发服务器

### 问题4: 类型错误

**症状**:
```
Type error: Cannot find module '@/xxx/xxx'
```

**解决方案**:
1. 检查 `tsconfig.json` 中的 paths 配置
2. 确保文件存在于正确位置
3. 重启TypeScript服务器:
   ```
   # VS Code中
   1. 按 Cmd + Shift + P (Mac) 或 Ctrl + Shift + P (Windows)
   2. 输入 "TypeScript: Restart TS Server"
   ```

### 问题5: 端口被占用

**症状**:
```
Error: Port 3000 is already in use
```

**解决方案**:
```bash
# 方案1: 杀死占用端口的进程
# Windows
netstat -ano | findstr :3000
taskkill /PID <PID> /F

# macOS/Linux
lsof -ti:3000 | xargs kill -9

# 方案2: 使用其他端口
npm run dev -- -p 3001
```

### 问题6: 数据库表不存在

**症状**:
```
Error: relation "users" does not exist
```

**解决方案**:
1. 确认已在Supabase中运行迁移SQL
2. 检查Supabase项目的Table Editor
3. 重新运行迁移脚本

### 问题7: API密钥无效

**症状**:
```
Error: OpenAI API key is invalid
```

**解决方案**:
1. 确认API密钥格式正确
2. 检查密钥是否过期（OpenAI密钥永不过期）
3. 检查账户是否有余额
4. 确认环境变量正确设置

### 问题8: 样式不生效

**症状**:
```
Tailwind CSS类名不生效，样式混乱
```

**解决方案**:
1. 确认已安装Tailwind CSS
2. 检查 `tailwind.config.ts` 中的 content 配置
3. 重启开发服务器
4. 清除浏览器缓存

#### Dockerfile

```dockerfile

COPY package*.json ./
RUN npm ci

# 构建应用
FROM base AS builder
WORKDIR /app

COPY . .
RUN npm run build

# 生产应用
FROM base AS runner
WORKDIR /app
ENV NODE_ENV production

RUN adduser --system --uid 1001 nextjs

COPY --from=builder --chown=nextjs:nodejs /app/.next/standalone ./
COPY --from=builder --chown=nextjs:nodejs /app/.next/static ./.next/static

USER nextjs
EXPOSE 3000

ENV PORT 3000
ENV HOSTNAME "0.0.0.0"

CMD ["node", "server.js"]
```

## 配置脚本

为了简化配置过程，可以创建以下辅助脚本：

### `scripts/setup.sh` (macOS/Linux)

```bash
#!/bin/bash

echo "🚀 开始配置日语口语练习App..."

# 检查Node.js
if ! command -v node &> /dev/null; then
    echo "❌ Node.js未安装"
    echo "请访问 https://nodejs.org/ 安装"
    exit 1
fi

NODE_VERSION=$(node -v)
echo "✅ Node.js版本: $NODE_VERSION"

# 检查npm
if ! command -v npm &> /dev/null; then
    echo "❌ npm未安装"
    exit 1
fi

# 安装依赖
echo "📦 安装依赖..."
npm install

# 检查.env.local
if [ ! -f .env.local ]; then
    echo "⚠️  .env.local 不存在"
    echo "从.env.example创建..."
    cp .env.example .env.local
    echo "⚠️  请编辑.env.local并填入您的API密钥"
fi

# 检查Supabase配置
if grep -q "your_supabase_url" .env.local; then
    echo "❌ 请先配置Supabase"
    echo "1. 访问 https://supabase.com 创建项目"
    echo "2. 获取API密钥"
    echo "3. 填入.env.local文件"
    exit 1
fi

echo "✅ 配置完成！"
echo "🎯 运行 'npm run dev' 启动开发服务器"
```

使用方法:
```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```
