## 🛠️ 开发指南

### 安装依赖

```bash
# 进入项目目录
cd SpeakJapaneseApp

# 安装依赖
npm install

# 或使用 yarn
yarn install

# 或使用 pnpm
pnpm install
```

### 配置Supabase

1. 在 [Supabase](https://supabase.com) 创建新项目
2. 运行SQL迁移文件：
   ```sql
   -- 复制 supabase/migrations/001_initial_schema.sql 的内容
   -- 在Supabase的SQL Editor中执行
   ```
3. 复制环境变量：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. 创建 `.env.local` 文件并添加变量

### 运行开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

### 关键修复记录

#### 修复1：模型文件名错误（2026-03-12）
**问题**：代码中硬编码为 `model.onnx`，实际文件为 `model.int8.onnx`

**修复文件**：
- `android/app/src/main/java/com/speakjapanese/app/plugins/LocalASRPlugin.java:50`
- `src/hooks/useLocalASR.ts:96`
- `src/services/asr/sherpa.service.ts:203`
- `src/plugins/local-asr/web.ts:22`

#### 修复2：浏览器环境兼容性（2026-03-12）
**问题**：`sherpa-onnx` 包包含 Node.js 代码，浏览器无法加载 `fs` 模块

**解决方案**：
1. `next.config.js` 添加 webpack fallback 配置
2. `sherpa.service.ts` 添加浏览器环境检查
3. 动态导入 sherpa-onnx（仅在客户端）

**配置**：
```javascript
webpack: (config, { isServer }) => {
  if (!isServer) {
    config.resolve.fallback = {
      ...config.resolve.fallback,
      fs: false,
      path: false,
      crypto: false,
    };
  }
  return config;
}
```
