# 日语口语练习Web应用 - 软件功能设计书（简化版）

## 1. 项目概述

### 1.1 项目名称
**日语口语练习Web应用 - 新概念日本语对话练习版**

### 1.2 项目背景
新概念日本语是一套经典的日语学习教材，共32课，包含丰富的实用对话场景。本Web应用旨在通过AI技术，帮助用户进行沉浸式的日语口语练习，提升日语口语表达能力。

### 1.3 项目目标
- 提供基于新概念日本语教材的对话练习功能
- 通过AI语音识别和智能评估，提供个性化学习反馈
- 打造沉浸式、互动式的日语学习体验
- 帮助用户从模仿到自由表达，逐步提升日语口语水平

### 1.4 平台定位
**Web优先**：专注于Web端开发，支持PC和移动浏览器访问

---

## 2. 目标用户

### 2.1 核心用户群体
- **初级日语学习者**：刚开始学习日语，需要基础对话练习
- **中级日语学习者**：有一定基础，希望提升口语流利度
- **日语考级备考者**：准备JLPT N5-N3级别的考试
- **日语爱好者**：对日本文化感兴趣，希望通过对话学习日语

### 2.2 用户痛点
- 缺乏日语对话练习机会
- 发音不准确，无人纠正
- 无法获得及时的口语反馈
- 学习内容枯燥，缺乏互动性
- 难以评估自己的口语水平

### 3.4 对话练习模块（核心功能）

#### 3.4.1 练习模式
**功能描述**：提供多种练习模式满足不同学习需求

**练习模式**：

**a) 标准练习模式**
- 逐句练习，系统给出中文提示
- 用户说出对应日文
- AI实时识别和评估
- 显示标准答案和发音要点

**b) 影子跟读模式**
- 先听标准发音
- 延迟1-2秒跟读
- 训练语调和节奏
- 对比展示

**c) 自由对话模式**
- 不限定台词，用户自由发挥
- AI根据语境判断回答是否合理
- 更注重流利度和自然表达

#### 3.4.2 练习界面设计

**界面布局**：
```
┌─────────────────────────────────┐
│  课程信息 + 进度条 + 设置按钮    │
├─────────────────────────────────┤
│                                  │
│     当前场景插图/动画            │
│                                  │
├─────────────────────────────────┤
│                                  │
│     AI角色头像 + 形象            │
│                                  │
├─────────────────────────────────┤
│                                  │
│     中文提示文字                 │
│     （您可以说的日文）           │
│                                  │
├─────────────────────────────────┤
│  [🔊 播放标准发音] [录音按钮]    │
│                                  │
├─────────────────────────────────┤
│     AI评估反馈区域               │
└─────────────────────────────────┘
```

**交互流程**：
1. 显示当前对话场景和角色
2. 用户点击播放按钮，听取标准发音（新增）
3. 系统给出中文提示（用户角色的台词含义）
4. 用户点击录音按钮开始说话
5. 实时显示音频波形
6. 再次点击按钮结束录音
7. 系统上传语音并进行识别
8. AI识别和分析语音内容
9. 显示评估结果和反馈
10. 用户点击"继续"进入下一句
11. 重复步骤2-10直到完成对话

**辅助功能**：
- 提示按钮：显示日文答案（使用次数限制）
- 跳过按钮：跳过当前句（记录为未完成）
- 暂停/继续练习
- 重新开始当前对话
- 查看完整对话文本

#### 3.4.3 语音朗读功能（新增）
**功能描述**：系统可以朗读对话内容，帮助用户学习

**详细功能**：
- **标准发音播放**：
  - 点击播放按钮，播放当前句子的标准日文发音
  - 使用TTS（文本转语音）技术
  - 支持调节播放速度（0.5x, 0.75x, 1.0x, 1.25x）
  - 支持循环播放
- **整段对话播放**：
  - 连续播放整个对话的所有句子
  - 自动播放当前角色或其他角色的台词
- **发音可视化**：
  - 显示音频波形
  - 显示播放进度
  - 高亮当前播放的句子

**技术实现**：
- 使用Web Speech API（浏览器原生TTS）
- 或集成第三方TTS服务：
  - Google Cloud Text-to-Speech（免费额度）
  - Amazon Polly（免费额度）
  - 微软Azure TTS（免费额度）

#### 3.4.4 语音录制功能

**技术要求**：

**录音操作**：

#### 3.4.5 AI语音识别功能
**功能描述**：将用户语音转换为文本并进行评估

**识别内容**：
- 语音转文字（ASR）
- 识别用户说的日文内容
- 提取发音特征

**技术方案**：
- 方案1：Web Speech API（浏览器原生，免费）
- 方案2：Google Cloud Speech-to-Text（免费额度：每月60分钟）
- 方案3：Azure Speech Services（免费额度：每月5小时）

**识别准确性优化**：
- 上下文关联识别
- 专业词汇库（新概念日本语词汇）

#### 3.4.6 AI智能评估系统（核心）
**功能描述**：利用大模型对用户口语进行多维度智能评估

**评估维度**：

**a) 准确性评估（40%权重）**
- 词汇正确性
- 语法正确性
- 句子完整性
- 与标准答案的匹配度

**b) 发音评估（30%权重）**
- 音素准确性
- 声调/重音
- 语调自然度
- 清晰度

**c) 流利度评估（15%权重）**
- 语速
- 停顿
- 连贯性

**d) 语气情感评估（10%权重）**
- 情感表达
- 自然度
- 敬语使用

**e) 自由度评估（5%权重，仅自由模式）**
- 创造性
- 丰富性

**评分机制**：
- 综合评分：0-100分
- 等级评定：
  - S级（90-100分）：完美
  - A级（80-89分）：优秀
  - B级（70-79分）：良好
  - C级（60-69分）：及格
  - D级（0-59分）：需改进

**大模型Prompt设计**：
```
你是一位专业的日语教学专家，负责评估用户的日语口语表达。

【任务】评估用户对以下日语句子的口语表达

【标准答案】：{standard_japanese}
【用户回答】：{user_response}
【对话场景】：{scene_context}
【角色信息】：{character_info}

【评估要求】
请从以下5个维度进行评估（每项0-100分）：
1. 准确性（词汇、语法、句子完整度）
2. 发音（音素、声调、语调、清晰度）
3. 流利度（语速、停顿、连贯性）
4. 语气情感（情感表达、自然度、敬语）
5. 自由度（创造性、丰富性、得体性）

【输出格式】
请以JSON格式输出评估结果：
{
  "overall_score": 综合评分,
  "dimension_scores": {
    "accuracy": 准确性得分,
    "pronunciation": 发音得分,
    "fluency": 流利度得分,
    "emotion": 语气情感得分,
    "freedom": 自由度得分
  },
  "grade": "等级(S/A/B/C/D)",
  "feedback": {
    "highlights": ["亮点1", "亮点2"],
    "issues": ["问题1", "问题2"],
    "suggestions": ["改进建议1", "改进建议2"]
  },
  "detailed_analysis": "详细分析（50字以内）"
}
```

**反馈展示**：
- 评分徽章（S/A/B/C/D等级图标）
- 评分进度条（各维度可视化）
- 高亮显示：
  - 绿色：正确部分
  - 黄色：需要改进部分
  - 红色：错误部分
- AI评语卡片
- 一键分享成绩

#### 5.1.4 AI技术
**语音识别（ASR）**
- **方案1：Web Speech API**（推荐，免费）
  - 浏览器原生支持
  - Chrome/Safari/Edge支持较好
  - 无需额外成本
  - 缺点：识别准确率可能不如商业API

- **方案2：Google Cloud Speech-to-Text**
  - 免费额度：每月60分钟
  - 识别准确率高
  - 支持日语
  - 超出后按量付费

- **方案3：Azure Speech Services**
  - 免费额度：每月5小时
  - 识别准确率高
  - 支持日语

**大模型（LLM）**
- **方案1：OpenAI GPT-4 / GPT-4o**
  - API调用
  - 按Token付费
  - 评估质量高

- **方案2：Anthropic Claude**
  - API调用
  - 按Token付费
  - 评估质量高

- **方案3：国内大模型**（更经济）
  - 阿里云通义千问
  - 百度文心一言
  - 智谱AI GLM
  - 免费额度更多

**语音合成（TTS）**
- **方案1：Web Speech API**（推荐，免费）
  - 浏览器原生支持
  - synthesis.speak()
  - 无需额外成本
  - 缺点：语音质量一般

- **方案2：Google Cloud Text-to-Speech**
  - 免费额度：每月100万字符
  - 语音质量高
  - 支持WaveNet语音

- **方案3：Amazon Polly**
  - 免费额度：每月500万字符
  - 语音质量高

- **方案4：微软Azure TTS**
  - 免费额度：每月500万字符
  - 语音质量高

**推荐组合**：
- MVP版本：Web Speech API（ASR + TTS）+ OpenAI/Claude（评估）
- 优化版本：Google/Azure ASR + 云TTS + LLM

#### 5.2.2 核心流程设计

**a) 用户注册登录流程**
```
用户输入邮箱和密码
   ↓
前端验证
   ↓
调用Supabase Auth注册/登录
   ↓
返回用户信息和Token
   ↓
存储到本地（LocalStorage/Cookie）
   ↓
重定向到首页
```

**b) 对话练习核心流程**
```
用户选择课程 → 加载课程数据（从Supabase）
   ↓
用户选择角色 → 初始化练习会话
   ↓
显示中文提示 → 用户点击播放TTS（朗读标准发音）
   ↓
用户点击录音 → 录制音频（Web Audio API）
   ↓
上传音频 → 服务器
   ↓
ASR识别 → 获取用户文本
   ↓
调用LLM评估 → 获取评分和反馈
   ↓
保存记录到Supabase → 返回结果
   ↓
展示反馈 → 下一句
```

**c) AI评估处理流程**
```
接收用户录音和标准答案
   ↓
ASR识别 → 获取用户文本
   ↓
准备评估上下文（场景、角色、对话）
   ↓
构造Prompt → 调用LLM API
   ↓
解析LLM响应 → 提取评分和反馈
   ↓
保存评估详情到Supabase → 返回给客户端
```
