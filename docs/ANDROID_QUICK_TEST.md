# Android ASR 快速测试指南

## 🚀 **快速编译**

### 方法 1: 使用脚本（推荐）

双击运行：
```
scripts\compile-android.bat
```

### 方法 2: 手动编译

```bash
# 1. 打开命令提示符（CMD）

# 2. 进入项目目录
cd C:\ClaudeCodeProject\SpeakJapaneseApp\android

# 3. 编译 APK
gradlew.bat assembleDebug

# 4. 检查结果
dir app\build\outputs\apk\debug\app-debug.apk
```

---

## 📱 **安装到设备**

### 方法 1: 使用 ADB

```bash
# 确保设备已连接
adb devices

# 安装 APK
adb install android\app\build\outputs\apk\debug\app-debug.apk
```

### 方法 2: 手动安装

1. 将 APK 文件复制到手机
2. 在手机上打开文件管理器
3. 找到 `app-debug.apk`
4. 点击安装
5. 允许未知来源安装（如果需要）

---

## 🧪 **测试功能**

### 启动应用

1. 在手机上打开"日语口语练习"应用
2. 应用会加载 Vercel 部署的 Web 界面
3. 等待页面加载完成

### 访问测试页面

在应用地址栏输入：
```
http://localhost:3000/test-android-asr
```

或导航到测试页面。

### 测试步骤

1. **初始化 ASR**
   - 点击"初始化 ASR"按钮
   - 等待 3-5 秒
   - 状态应显示"就绪"

2. **开始录音**
   - 点击"开始录音"按钮
   - 允许麦克风权限
   - 说日语句子（如"こんにちは"）

3. **停止录音**
   - 点击"停止录音"按钮
   - 等待识别结果（< 1秒）
   - 查看识别的文本

## 🐛 **常见问题**

### 问题 1: APK 编译失败

**症状**: 编译出现错误

**解决方案**:
1. 检查 Java JDK 是否安装（需要 JDK 11 或更高）
2. 检查 Android SDK 是否安装
3. 清理并重新编译：`gradlew.bat clean assembleDebug`

### 问题 2: 应用无法安装

**症状**: 安装失败或提示版本不兼容

**解决方案**:
1. 确认 Android 版本 ≥ 5.0 (API 21)
2. 卸载旧版本应用
3. 重新安装

### 问题 3: ASR 初始化失败

**症状**: 状态始终显示"未初始化"

**解决方案**:
1. 查看日志：`adb logcat | findstr "LocalASR"`
2. 确认模型文件存在
3. 检查是否有足够存储空间

### 问题 4: 识别结果为空

**症状**: 录音正常但无识别结果

**解决方案**:
1. 确认在安静环境测试
2. 说话清晰、大声
3. 检查麦克风权限
