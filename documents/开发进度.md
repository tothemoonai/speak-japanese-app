## 🎯 最新更新 (2025-03-15 深夜 - 第七次)

### ✅ Bug修复 - 用户等级显示 & Android TTS支持

**用户反馈问题**:
1. ❌ 用户等级显示不正常
2. ❌ Android手机听示范发音显示"浏览器不支持语音播放"

**问题1：用户等级显示不正常**

**根本原因**:
- 用户数据存储在多个位置：`user.level`、`user.user_metadata.level`
- 有些地方优先读取了错误的位置导致显示不正确

**解决方案**:
- ✅ 创建辅助函数 `getUserLevel()` 自动从多个位置获取等级
- ✅ 更新所有显示用户等级的地方使用此函数
- ✅ 同时修复昵称显示问题（`user.user_metadata.nickname`）

**新增文件**:
- `src/lib/utils/user.ts` - 用户数据辅助函数

**修改文件**:
- `src/app/dashboard/page.tsx` - 使用getUserLevel函数
- `src/app/settings/page.tsx` - 使用getUserLevel函数

**代码改进**:
```typescript
// Before (可能失败)
{user.level === 'beginner' && '初级'}

// After (自动获取)
import { getUserLevel } from '@/lib/utils/user';
{getUserLevel(user) === 'beginner' && '初级'}
```

**问题2：Android TTS不支持**

**根本原因**:
- Android WebView的Web Speech API支持需要初始化时间
- `speechSynthesis.getVoices()` 在Android上可能返回空数组
- 需要监听 `voiceschanged` 事件

**解决方案**:
- ✅ 添加voices加载状态检测
- ✅ 监听 `onvoiceschanged` 事件
- ✅ 显示"正在加载语音..."提示
- ✅ voices加载完成后才启用播放按钮
- ✅ 改进错误处理和日志

**技术实现**:
```typescript
// 检查voices是否可用
const loadVoices = () => {
  const voices = window.speechSynthesis.getVoices();
  if (voices && voices.length > 0) {
    setVoicesLoaded(true);
    setIsSupported(true);
  }
};

// 立即加载
loadVoices();

// 监听voiceschanged事件（Android需要）
window.speechSynthesis.onvoiceschanged = loadVoices;
```

**用户体验改进**:
- 📱 Android用户现在可以正常播放示范发音
- ⏳ 显示加载状态而不是直接显示"不支持"
- 🎯 更准确的错误提示
- 🔄 自动重试机制

- ❌ **方法签名不匹配**：`LocalASRPlugin.java` 中的 `recognize()` 方法（第179行）调用了 `OfflineStream.acceptWaveform()`

- ✅ **新方法已存在**：已经实现了正确的 `startRecording()` 和 `stopAndRecognize()` 方法

**错误代码位置：**
```java
// ❌ 第 155-195 行：旧的 recognize() 方法
@PluginMethod
public void recognize(PluginCall call) {
    // ...
    OfflineStream stream = recognizer.createStream();
    stream.acceptWaveform(samples, sampleRate);  // ❌ JNI 方法不存在！
    // ...
}
```

### 🔧 **修复方案**

**1. 移除旧的不兼容方法**
- 删除 `recognize()` 方法（第 155-195 行）
- 保留新的 `startRecording()` 和 `stopAndRecognize()` 方法

```javascript
webpack: (config, { isServer }) => {
  if (!isServer) {
    config.resolve.fallback = {
      ...config.resolve.fallback,
      fs: false,
      path: false,
      crypto: false,
    };
  }
  return config;
}
```
